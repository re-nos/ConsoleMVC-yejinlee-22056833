# 콘솔 UI 구성 (수정본 정리)

사용자가 수정한 화면 구성을 정리한 문서입니다. 이 문서를 기준으로
`console_view.py` / `menu_view.py`에 반영합니다.

## 공통 규칙

- 구분선: `=`(굵은 구분선, 71자)은 화면 최상단/타이틀에, `-`(얇은 구분선, 71자)은
  섹션 내부 구분에 사용합니다.
- 메뉴 항목은 `[번호] 항목명` 형식으로 표기합니다.
- 승인/거절과 같은 Yes/No 선택은 `[Y]` / `[N]` 형식으로 표기합니다 (기존 `a`/`r` → 변경).
- 표(테이블)는 헤더 아래에 얇은 구분선을 추가합니다.
- 메인 메뉴 상단에 시스템 현황 요약(날짜/시간, 등록 시료 수, 총 재고, 전체 주문 건수,
  생산 대기 건수)을 표시합니다.

### 요약 통계 데이터 소스 (구현 참고용)

| 표시 항목       | 계산 방법                                                        |
|-----------------|-------------------------------------------------------------------|
| 등록 시료 n종   | `len(sample_controller.list_samples())`                            |
| 총 재고 n ea    | `sum(s.stock for s in sample_controller.list_samples())`           |
| 전체 주문 n건   | `sum(monitoring_controller.count_by_status().values())` (REJECTED 제외) |
| 생산라인 n건 대기 | `len(production_controller.waiting_jobs())`                     |
| 날짜/시간       | 화면 렌더링 시점의 로컬 날짜/시간                                 |

## 1. 메인 메뉴

```
==================== S-Semi 생산주문관리 시스템 ====================

등록 시료   {sample_count}종      총 재고    {total_stock} ea
전체 주문   {total_order_count}건      생산라인    {waiting_count}건 대기
---------------------------------------------------------------------
[1] 시료 관리               [2] 시료 주문
[3] 주문 승인/거절           [4] 모니터링
[5] 생산 라인               [6] 출고 처리
[0] 종료
---------------------------------------------------------------------
메뉴 선택>
```

## 2. 시료 관리

### 2-1. 서브메뉴

```
================================================
[시료 관리]
---------------------------------------------------------------------
[1] 등록  [2] 목록  [3] 이름 검색
---------------------------------------------------------------------
선택>
```

### 2-2. 등록 (`[1]`)

```
시료 ID:
시료 이름:
평균 생산시간:
수율(0~1):
시료가 등록되었습니다: {sample.id}
```

### 2-3. 목록 / 검색 결과 (`[2]`, `[3]`)

```
등록 시료 목록 (총 {sample_count}종)
ID      이름            평균생산시간수율    재고
---------------------------------------------------------------------
S1      SiC Wafer       10.0    0.5     0
```

빈 목록: `등록된 시료가 없습니다.`

## 6. 생산 라인

### 6-1. 현재 생산 중

```
[생산 중] 주문ID={job.order_id} 시료ID={job.sample_id} 생산수량={job.quantity_to_produce} 남은시간={job.remaining_time}
```

미가동: `현재 생산 중인 작업이 없습니다.`

### 6-2. 대기열

```
[생산 대기열]
  주문ID={job.order_id} 시료ID={job.sample_id} 생산수량={job.quantity_to_produce} 총생산시간={job.total_time}
```

대기열 없음: `대기 중인 생산 작업이 없습니다.`

### 6-3. 진행 시간 입력 및 완료 메시지

```
생산 진행 시간(분, 0=진행 안 함):
주문 {order.id}의 생산이 완료되어 CONFIRMED로 전환되었습니다.
```

## 변경 요약 (기존 → 수정본)

- 메인 메뉴에 날짜/시간 및 시스템 현황 요약(등록 시료 수/총 재고/전체 주문/생산 대기) 추가
- 메뉴 항목 표기를 `1. 항목` → `[1] 항목` 브래킷 형식으로 변경, 2열 배치 적용
- 서브메뉴(`[시료 관리]`)에 타이틀 구분선과 브래킷 메뉴 표기 추가
- 승인/거절 입력을 `a`/`r` → `[Y]`/`[N]` 표기로 변경
- 시료 목록 상단에 "등록 시료 목록 (총 n종)" 타이틀 추가
- 모든 표(주문, 시료, 재고 현황)에 헤더 아래 구분선(`-`) 추가

## 반영 예정 작업

1. `console_view.py`: 표 헤더 아래 구분선 출력 로직, 서브메뉴 타이틀 헬퍼 추가
2. `menu_view.py`: 메인 메뉴 렌더링 시 요약 통계 계산(위 표 참고) 후 배너에 삽입,
   승인/거절 입력을 `Y`/`N`(대소문자 무관)으로 변경
3. `python -m console_mvc.app`으로 직접 실행해 새 화면 확인 후 커밋
